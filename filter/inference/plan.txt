CANONICAL INTER-SERVICE CONTRACTS (LOCK THESE)
----------------------------------------------

Contract A: Webhooks -> Filter
------------------------------
Transport:
    gRPC ClassifyMessage (fire-and-forget in current hot path).

Payload shape (stringified JSON envelope):
    {
        "meta": {
            "user_id": "...",
            "company_id": 123,
            "source": "slack|gmail",
            "sent_at": "ISO-8601",
            ... optional fields ...
        },
        "text": "raw message text"
    }

Rule:
    Filter must classify the "text" field, NOT the raw JSON blob.


Contract B: Filter gRPC Surface
-------------------------------
Must remain stable:
    - ClassifyMessage(ClassifyRequest) -> ClassifyResponse
    - ClassifyMessages(BatchClassifyRequest) -> BatchClassifyResponse

Batch behavior:
    - Preserve order
    - Partial failures produce safe default response entries
    - Empty list returns empty results


Contract C: Timestamp Semantics
-------------------------------
Inference input should include temporal context as prefix:
    [YYYY-MM-DD HH:MM] <message>

Timestamp source priority:
    1) envelope meta.sent_at (preferred)
    2) server current time fallback


Contract D: Ownership Boundaries
--------------------------------
Current DEV path:
    webhooks dispatches to filter; downstream pipeline handles persistence/escalation.

Do not reintroduce old direct DB writes in webhooks filter gate path.
