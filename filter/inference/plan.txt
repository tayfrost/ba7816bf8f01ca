SentinelAI Filter Inference Service - Implementation Plan
===============================================================

GOAL: gRPC inference service for SentiBERT with monitoring & optimization

─────────────────────────────────────────────────────────────────

1. PROTO SCHEMA DEFINITION
   [ ] Create filter.proto in protos/
       - ClassifyMessage RPC
       - ClassifyRequest: { string message }
       - ClassifyResponse: { 
           string category, float category_confidence,
           string severity, float severity_confidence,
           bool is_risk
         }

2. COMPILE PROTO FILES
   [ ] Generate Python code:
       python -m grpc_tools.protoc -I./protos \
         --python_out=./generated \
         --grpc_python_out=./generated \
         ./protos/filter.proto
   [ ] Creates: filter_pb2.py, filter_pb2_grpc.py

3. IMPLEMENT INFERENCE ENGINE
   [ ] Create engine.py:
       - InferenceEngine class
       - Load model via model_factory.load_production_model()
       - predict(text) -> (category, severity, scores)
       - Singleton pattern for model caching

4. IMPLEMENT GRPC SERVER
   [ ] Create server.py:
       - FilterServiceServicer implements generated stub
       - ClassifyMessage handler calls engine.predict()
       - Error handling (INVALID_ARGUMENT, UNAVAILABLE)
       - Startup: load model, bind port 50051
   [ ] Test with sample client
   [ ] Verify predictions match evaluation results

5. TEXT CHUNKING FOR LONG MESSAGES
   [ ] Implement text splitting in engine.py:
       - Split messages >128 tokens into chunks
       - Sliding window or sentence-based splitting
       - Aggregate predictions (max risk, weighted avg, etc.)
   [ ] Test with long messages (>200 tokens)

6. PROMETHEUS METRICS
   [ ] Add prometheus_client dependency
   [ ] Instrument metrics:
       - request_count (Counter)
       - request_latency_seconds (Histogram with p50/p95/p99)
       - requests_in_flight (Gauge)
       - model_inference_seconds (Histogram)
   [ ] Expose /metrics HTTP endpoint (port 8000)

7. MODEL QUANTIZATION & BENCHMARKING
   [ ] Implement INT8 quantization:
       - Use torch.quantization.quantize_dynamic
       - Compare FP32 vs INT8 latency
   [ ] Benchmark both versions:
       - Single request latency
       - Throughput (requests/sec)
       - Memory usage
   [ ] Document results and select optimal version

─────────────────────────────────────────────────────────────────
